title: "AgentKit 命令兼容"
---

VeADK 命令行内置了 AgentKit 命令。在任意 AgentKit 命令前加上 `veadk agentkit` 前缀，即可复用熟悉的 AgentKit 工作流来创建、构建、部署并管理火山引擎 AgentKit 平台上的项目。

```bash
veadk agentkit --help
```

<Callout type="info">
  使用前请确保已安装 VeADK，并在项目根目录的 `.env` 文件中配置好火山引擎访问凭据。
</Callout>

## 命令一览

| 命令 | 描述 |
| :-- | :-- |
| `veadk agentkit init` | 用模板创建新项目，或包装一个已有的 Agent。 |
| `veadk agentkit config` | 配置 AgentKit（支持交互与非交互模式）。 |
| `veadk agentkit build` | 为 Agent 构建 Docker 镜像。 |
| `veadk agentkit deploy` | 将 Agent 部署到目标环境。 |
| `veadk agentkit launch` | 一条命令完成构建与部署。 |
| `veadk agentkit invoke` | 向已部署的 Agent 发送测试请求。 |
| `veadk agentkit status` | 查看 Agent 运行时状态。 |
| `veadk agentkit destroy` | 销毁正在运行的 Agent 运行时。 |
| `veadk agentkit memory` | 管理 AgentKit Memory 集合。 |
| `veadk agentkit knowledge` | 管理 AgentKit 知识库。 |
| `veadk agentkit tools` | 管理 AgentKit 工具与会话。 |
| `veadk agentkit runtime` | 管理 AgentKit 运行时。 |

## 常用流程

创建项目，然后一步构建并部署：

```bash
veadk agentkit init
veadk agentkit launch
```

## 调用 HarnessApp Runtime

`veadk agentkit invoke` 可以直接调用 HarnessApp Runtime，并通过请求参数临时开启 Harness Extension。常用参数如下：

| 参数 | 说明 |
| :-- | :-- |
| `--harness` | HarnessApp 中的智能体名称。 |
| `--endpoint` | Runtime endpoint。 |
| `--apikey` | Runtime API Key。 |
| `--model-id` | 单次调用覆盖模型名称。 |
| `--tools` | 单次调用覆盖工具列表，逗号分隔。 |
| `--skills` | 单次调用覆盖技能列表，逗号分隔。 |
| `--enable-harness-enhance` | 本次调用启用 Harness Extension。 |
| `--harness-components` | 本次调用启用的 Harness 组件，逗号分隔。 |
| `--harness-profile` | 可选运行 profile。 |
| `--harness-compression-provider` | 可选压缩 provider，默认 `builtin`。 |

```bash
veadk agentkit invoke \
  --harness research-agent \
  --endpoint "$HARNESS_URL" \
  --apikey "$HARNESS_KEY" \
  --model-id "your-model-id" \
  --tools web_search,run_code \
  --enable-harness-enhance \
  --harness-components "invocation_context,compactor,response_verification" \
  "Summarize the tool results with evidence."
```
