---
title: "运行时动态配置"
---

环境变量与默认值在启动时确定，进程运行后不再改变。生产环境往往需要在不重启、不重新部署的前提下调整智能体，例如临时更换提示词或切换模型对比效果。运行时动态配置（热更新）即为此而设。

## 工作方式

可变设定托管在配置中心，运转分三步：

1. **下发**：将描述智能体设定的配置发布到配置中心；
2. **订阅**：运行中的智能体订阅该配置；
3. **生效**：配置变化时，智能体实时拉取并就地更新，无需重启。

底层由火山引擎托管配置服务承载，凭据通过环境变量提供，或在云上部署时由平台自动注入，无需关心连接细节。

<Callout type="info">
动态配置适合调整频繁变动的内容（提示词、模型）。密钥与连接信息等启动期配置仍由[环境变量](/cn/docs/references/configuration/environment-variables)管理。
</Callout>

## 可热更新的字段

配置采用 JSON 结构，用一个 `agent` 数组描述一个或多个智能体的设定。每个条目可包含：

| 字段 | 说明 |
| :- | :- |
| `id` | 目标智能体的标识，须与运行中的智能体一致，用于精确匹配 |
| `name` | 名称（用于展示与日志） |
| `description` | 描述（用于展示与日志） |
| `model_name` | 模型名称，变更后智能体会切换到新模型 |
| `instruction` | 系统提示词，定义智能体的行为 |

```json
{
  "agent": [
    {
      "id": "...",
      "name": "...",
      "description": "...",
      "model_name": "...",
      "instruction": "..."
    }
  ]
}
```

更新通过 `id` 匹配到对应的智能体；命中后，名称、描述、模型与提示词会被即时替换——其中模型变更会触发一次平滑的模型切换。
