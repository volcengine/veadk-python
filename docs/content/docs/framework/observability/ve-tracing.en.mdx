## CozeLoop Platform

After connecting a Volcengine agent developed with VeADK to CozeLoop, you can use CozeLoop's evaluation feature to evaluate the agent, or use the Trace feature for trace observation. The trace data of a Volcengine agent can be reported directly to CozeLoop for trace observation; agents registered in CozeLoop can also be evaluated through the observation feature.

### Preparation

Fill in the cozeloop attributes under the `observability` field of the VeADK configuration file `config.yaml`. For detailed descriptions and examples of the configuration file, refer to the configuration file documentation.

- endpoint: Fixed to `https://api.coze.cn/v1/loop/opentelemetry/v1/traces`
- api_key: The CozeLoop access key. Personal access tokens, OAuth access tokens, and service access tokens are supported. For how to obtain one, see [Configure a Personal Access Token](https://loop.coze.cn/open/docs/cozeloop/authentication-for-sdk#05d27a86).
- service_name: The ID of the CozeLoop workspace. After logging into CozeLoop, switch in the top-left corner to the workspace where you want to store the Volcengine agent data, and obtain the workspace ID from the part of the URL after the `space` keyword. For example, in `https://loop.coze.cn/console/enterprise/personal/space/739XXXXXXXX092/pe/prompts`, **739XXXXXXXX092** is the workspace ID.

![CozeLoop workspace](/assets/images/observation/coze-spaceid.png)

```yaml title="config.yaml"
model:
  agent:
    provider: openai
    name: doubao-seed-1-6-flash-250828
    api_base: https://ark.cn-beijing.volces.com/api/v3
    api_key: your api_key

volcengine:
  access_key: your ak
  secret_key: your sk

observability:
  opentelemetry:
    cozeloop:
      endpoint: https://api.coze.cn/v1/loop/opentelemetry/v1/traces
      api_key: your your api_key
      service_name: your cozeloop space id
```

### Deploy and Run

#### CozeLoop Exporter Integration Code

```python title="agent.py"
import asyncio

from veadk import Agent, Runner
from veadk.memory.short_term_memory import ShortTermMemory
from veadk.tools.demo_tools import get_city_weather
from veadk.tracing.telemetry.exporters.cozeloop_exporter import CozeloopExporter
from veadk.tracing.telemetry.opentelemetry_tracer import OpentelemetryTracer
from veadk.tracing.telemetry.exporters.cozeloop_exporter import CozeloopExporterConfig


cozeloop_exporter = CozeloopExporter()

exporters = [cozeloop_exporter]
tracer = OpentelemetryTracer(exporters=exporters)

agent = Agent(tools=[get_city_weather], tracers=[tracer])

session_id = "session_id_pj"

runner = Runner(agent=agent, short_term_memory=ShortTermMemory())

prompt = "How is the weather like in Beijing? Besides, tell me which tool you invoked."

asyncio.run(runner.run(messages=prompt, session_id=session_id))
```

```bash

```

## APMPlus Platform

A Volcengine agent developed with VeADK can be connected to the Volcengine APMPlus platform by defining an APMPlus data exporter, enabling trace observation.

### Preparation

- endpoint: Set the APMPlus access point to `http://apmplus-cn-beijing.volces.com:4317`
- api_key: A valid application key must be filled in.
- service_name: Specify the service name, which can be modified as needed.

Initialize the APMPlusExporter: use APMPlusExporterConfig to configure the endpoint, application key, and service name, then create an APMPlusExporter instance with the configuration obtained from environment variables. Sample code is as follows:

```yaml title="config.yaml"
model:
  agent:
    provider: openai
    name: doubao-seed-1-6-flash-250828
    api_base: https://ark.cn-beijing.volces.com/api/v3
    api_key: your api_key

volcengine:
  access_key: your ak
  secret_key: your sk

observability:
  opentelemetry:
    apmplus:
      endpoint: http://apmplus-cn-beijing.volces.com:4317
      api_key: your api_key
      service_name: apmplus_veadk_pj
```

```python title="agent.py"
import asyncio

from veadk import Agent, Runner
from veadk.memory.short_term_memory import ShortTermMemory
from veadk.tools.demo_tools import get_city_weather
from veadk.tracing.telemetry.exporters.apmplus_exporter import APMPlusExporter
from veadk.tracing.telemetry.opentelemetry_tracer import OpentelemetryTracer
from veadk.tracing.telemetry.exporters.apmplus_exporter import APMPlusExporterConfig
from os import getenv


apmplus_exporter = APMPlusExporter()

exporters = [apmplus_exporter]

tracer = OpentelemetryTracer(exporters=exporters)

agent = Agent(tools=[get_city_weather], tracers=[tracer])

session_id = "session_id_pj"

runner = Runner(agent=agent, short_term_memory=ShortTermMemory())

prompt = "How is the weather like in Beijing? Besides, tell me which tool you invoked."
asyncio.run(runner.run(messages=prompt, session_id=session_id))
```

### Deploy and Run

Run the `agent.py` code above locally to trigger the APMPlus tracer to record the invocation of each node of the agent's execution, and upload metrics information to cloud storage:

```bash
python agent.py
```

## TLS Platform

A Volcengine agent developed with VeADK can be connected to Volcengine Log Service (TLS) by defining a TLS data exporter, and you can observe the agent's execution traces in the TLS observation module.

### Preparation

#### Create a Tracing Project and Instance in VeADK Code

```yaml title="config.yaml"
model:
  agent:
    provider: openai
    name: doubao-seed-1-6-flash-250828
    api_base: https://ark.cn-beijing.volces.com/api/v3
    api_key: your api_key

volcengine:
  access_key: your ak
  secret_key: your sk

observability:
  opentelemetry:
    tls:
      endpoint: https://tls-cn-beijing.volces.com:4318/v1/traces
      service_name: tp_pj
      region: cn-beijing
```

```python title="agent.py"
import asyncio

from veadk import Agent, Runner
from veadk.memory.short_term_memory import ShortTermMemory
from veadk.tools.demo_tools import get_city_weather
from veadk.tracing.telemetry.exporters.tls_exporter import TLSExporter
from veadk.tracing.telemetry.opentelemetry_tracer import OpentelemetryTracer
from veadk.tracing.telemetry.exporters.tls_exporter import TLSExporterConfig
from veadk.integrations.ve_tls.ve_tls import VeTLS
from os import getenv

# Initialize the VeTLS client to create the log project and tracing instance
ve_tls_client = VeTLS()

# Create the log project
project_name = "veadk_pj"
log_project_id = ve_tls_client.create_log_project(project_name)
print(f"Created log project with ID: {log_project_id}")

# Create the tracing instance
trace_instance_name = getenv("OBSERVABILITY_OPENTELEMETRY_TLS_SERVICE_NAME")
trace_instance = ve_tls_client.create_tracing_instance(log_project_id, trace_instance_name)
print(f"Created trace instance with ID: {trace_instance['TraceInstanceId']}")
```

#### TLS Exporter Integration Code Example

```python title="agent.py"
import asyncio

from veadk import Agent, Runner
from veadk.memory.short_term_memory import ShortTermMemory
from veadk.tools.demo_tools import get_city_weather
from veadk.tracing.telemetry.exporters.tls_exporter import TLSExporter
from veadk.tracing.telemetry.opentelemetry_tracer import OpentelemetryTracer
from veadk.tracing.telemetry.exporters.tls_exporter import TLSExporterConfig
from veadk.integrations.ve_tls.ve_tls import VeTLS
from os import getenv

# Initialize the TLSExporter to report tracing data
tls_exporter = TLSExporter(
    config=TLSExporterConfig(
        topic_id=trace_instance.get('TraceTopicId', trace_instance_name),
    )
)

exporters = [tls_exporter]

tracer = OpentelemetryTracer(exporters=exporters)

agent = Agent(tools=[get_city_weather], tracers=[tracer])

session_id = "session_id_pj"

runner = Runner(agent=agent, short_term_memory=ShortTermMemory())

prompt = "How is the weather like in Beijing? Besides, tell me which tool you invoked."

asyncio.run(runner.run(messages=prompt, session_id=session_id))
```

### Deploy and Run

Run the `agent.py` code above locally to trigger the creation of the TLS Project and Topic, and record the invocation of each node of the agent's execution through the tracer:

```bash
python agent.py
```
