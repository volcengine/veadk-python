Observability is one of the core capabilities of enterprise-grade agent applications. By recording the full chain of an agent's execution, it helps developers understand the agent's internal behavior, locate issues, analyze performance, and continuously optimize.

## What Is Observability

In an agent system, observability refers to the **full-chain recording and tracing** of the agent's execution, from receiving a user request to generating the final response. This includes:

- The receipt time and content of user input and requests;
- The agent's internal reasoning process, such as model calls, tool execution, and policy decisions;
- The invocation records of each tool, service, or external system interface;
- The generation of responses and the output results.

With these records, developers can turn the agent's execution process into structured data that is analyzable and traceable.

## Core Capabilities

VeADK's observability follows the [OpenTelemetry](https://opentelemetry.io/docs/specs/semconv/registry/attributes/gen-ai/) generative-AI semantic conventions and mainly provides the following capabilities:

- **Visualized traces**: The complete flow of each request, from intake to response, can be traced;
- **Structured data**: Records context, state, event types, latency, and more;
- **Cross-component tracing**: Covers the agent, tools, memory modules, knowledge bases, and external interfaces;
- **Multi-agent coordination**: Analyzes the invocation relationships and data flow between agents in multi-agent collaboration scenarios.
