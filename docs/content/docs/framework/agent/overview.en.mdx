---
title: "Core concepts"
description: "What an agent is, and its four building blocks: model, tools, memory, and knowledge base."
---

An agent is a software entity with a large model as its decision engine: it understands input, plans steps, calls tools, reads and writes memory and knowledge, and drives a task from understanding to completion on its own. Agents can also collaborate over the [A2A protocol](/en/docs/framework/agent/multi-agent/a2a) to form more capable systems.

A VeADK agent is made of four building blocks:

![Agent architecture overview](/assets/images/overview/agent-overview.png)

## Model

The agent's "brain" — a large language model (LLM), vision-language model (VLM/MLLM), and so on. It understands natural-language or multimodal input, and plans steps and tool calls. See model configuration in [Create an agent](/en/docs/framework/agent/create).

## Tools

Tools are the agent's "hands" — they perform the concrete actions the model plans and fetch real-time data (the model can't check the weather, book a flight, or query a database on its own).

| Type | Examples | Description |
| :--- | :--- | :--- |
| Information | `web_search`, `vesearch` | Web search, vector retrieval |
| System actions | API calls, DB queries, local functions | Call services, query data, run code |
| Standard protocols | `MCP`, function calling | Standardize model–tool interaction |

See [Built-in tools](/en/docs/framework/tools/builtin) and [Custom function tools](/en/docs/framework/tools/custom-function).

## Memory

Memory keeps conversational context and user preferences, making interactions coherent and personalized.

| Type | Description | Storage |
| :--- | :--- | :--- |
| [Short-term](/en/docs/framework/memory/short-term) | Context within a single session | Local memory, MySQL |
| [Long-term](/en/docs/framework/memory/long-term) | History and preferences across sessions | Cloud Search (OpenSearch), Redis, Viking memory |

## Knowledge base

A knowledge base adds private / domain knowledge to the model via RAG (retrieval-augmented generation): it retrieves relevant passages first, then has the model answer from them, reducing hallucination. It supports auto-chunking of TXT, PDF, Word, and more, stored in the Viking knowledge base, Cloud Search, etc. See [Knowledge base](/en/docs/framework/knowledgebase/overview).
