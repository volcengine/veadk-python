---
title: "核心概念"
description: "智能体（Agent）的基本概念，以及它的四个核心要素：模型、工具、记忆、知识库。"
---

智能体（Agent）是以大模型为决策引擎的软件实体：它理解输入、规划步骤、调用工具、读写记忆与知识库，自主地把任务从理解推进到完成。智能体之间还能通过 [A2A 协议](/cn/docs/framework/agent/multi-agent/a2a)协作，组成更强的系统。

VeADK 的智能体由四个核心要素组成：

![Agent 架构概览](/assets/images/overview/agent-overview.png)

## 模型（Model）

智能体的“大脑”——大语言模型（LLM）、视觉语言模型（VLM/MLLM）等。它理解自然语言或多模态输入，负责解析需求、规划步骤、决定何时调用工具。参见[构建智能体](/cn/docs/framework/agent/create)中的模型配置。

## 工具（Tools）

工具是智能体的“手脚”，执行模型规划出的具体动作、获取实时数据（模型本身无法查天气、订机票或查数据库）。

| 类型 | 示例 | 说明 |
| :--- | :--- | :--- |
| 信息获取 | `web_search`、`vesearch` | 联网搜索、向量检索 |
| 系统操作 | API 调用、数据库查询、本地函数 | 调用外部服务、查询数据、执行代码 |
| 标准协议 | `MCP`、Function Call | 规范模型与工具的交互 |

参见[内置工具](/cn/docs/framework/tools/builtin)与[自定义函数工具](/cn/docs/framework/tools/custom-function)。

## 记忆（Memory）

记忆保存对话上下文与用户偏好，让交互更连贯、更个性化。

| 类型 | 说明 | 存储 |
| :--- | :--- | :--- |
| [短期记忆](/cn/docs/framework/memory/short-term) | 单会话内的上下文 | 本地内存、MySQL |
| [长期记忆](/cn/docs/framework/memory/long-term) | 跨会话的历史与偏好 | 云搜索（OpenSearch）、Redis、Viking 记忆库 |

## 知识库（Knowledge Base）

知识库通过 RAG（检索增强生成）为模型补充私有 / 专业知识，先检索相关片段再交给模型作答，从而减少幻觉。支持 TXT、PDF、Word 等文档自动分片，存储于 Viking 知识库、云搜索等。参见[知识库](/cn/docs/framework/knowledgebase/overview)。
