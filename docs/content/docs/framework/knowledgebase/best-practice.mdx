本文把知识库与多智能体结合，构建一个 IT 运维助手：一个总控 Agent 把用户问题路由到对应的部门子 Agent（网络、主机、应用、安全），每个子 Agent 只依据自己的知识库（运维手册 / Runbook）作答。

## 场景

- **跨部门知识整合**：整合网络、服务器、应用、安全等部门的 Runbook；
- **意图路由**：总控 Agent 判断问题归属，调用相应子 Agent；
- **动作需确认**：涉及创建工单等动作时，先征得用户确认再调用工具。

<Callout type="info">
示例用知识库的 `local` 后端，便于本地跑通；它会对文档做向量化，需要安装 `pip install "veadk-python[extensions]"`。生产环境请改用 `viking` 或 `opensearch`。
</Callout>

## 第 1 步：准备知识库

把各部门的运维手册整理成 Markdown。示例（节选）：

```markdown title="network_sop.md"
# 网络排障 Runbook（摘要）

## DNS 解析异常

- 检查本机 /etc/resolv.conf；
- 检查上游 DNS 可达性；
- 刷新 DNS 缓存：`systemd-resolve --flush-caches`；
- 如为权威解析变更：确认变更已在权威 DNS 生效，等待 TTL。
```

```markdown title="host_sop.md"
# 主机/中间件 Runbook（摘要）

## CPU 飙高

- top 确认进程；
- 采集 dump / 火焰图；
- 参考 SOP：collect-cpu-profile。
```

```markdown title="app_sop.md"
# 业务应用 Runbook（摘要）

## 502 网关错误

- 查看 Nginx/Ingress 错误日志；
- 回滚至上一稳定版本；
- 触发金丝雀放量回退。
```

```markdown title="security_sop.md"
# 安全事件 Runbook（摘要）

## 失陷主机初步处置

- 下线隔离；
- 采集 Hash / IOC；
- 通报安全应急群。
```

为每个部门各建一个知识库（用不同的 `index` 隔离），分别注入对应文档：

```python
from veadk.knowledgebase import KnowledgeBase


def build_kb(index: str, files: list[str]) -> KnowledgeBase:
    kb = KnowledgeBase(backend="local", index=index)
    kb.add_from_files(files)
    return kb


kb_network = build_kb("itops_network", ["network_sop.md"])
kb_host = build_kb("itops_host", ["host_sop.md"])
kb_app = build_kb("itops_app", ["app_sop.md"])
kb_sec = build_kb("itops_security", ["security_sop.md"])
```

<Callout type="info">
`KnowledgeBase` 用 `index` 标识知识库（也可传 `app_name`，缺省时会回退为 `index`）。每个子 Agent 挂自己的 `index`，实现部门级知识隔离。
</Callout>

## 第 2 步：定义动作工具

总控 Agent 可调用两个工具——查询 CMDB 与创建工单（这里用内存数据模拟）：

```python
def query_cmdb(hostname: str) -> str:
    """根据主机名查询 CMDB 中的资产信息。

    Args:
        hostname: 主机名，如 "web-01"。

    Returns:
        资产信息描述。
    """
    cmdb = {
        "web-01": "应用服务器，归属：应用部，IP：192.0.2.21，状态：在线",
        "db-01": "数据库服务器，归属：主机部，IP：192.0.2.11，状态：在线",
    }
    return cmdb.get(hostname, f"未在 CMDB 中找到主机 {hostname}")


def create_ticket(title: str, detail: str) -> str:
    """创建一条运维工单。

    Args:
        title: 工单标题。
        detail: 工单详情。

    Returns:
        新建工单的编号与状态。
    """
    ticket_id = "T-2001"  # 示例：真实场景对接工单系统
    return f"已创建工单 {ticket_id}：{title}（{detail}），状态：待处理"
```

## 第 3 步：构建多智能体并挂载知识库

每个子 Agent 挂一个部门知识库，总控 Agent 用 `sub_agents` 编排，并持有动作工具：

```python
from veadk import Agent, Runner

MODEL_NAME = "doubao-seed-1-8-251228"

network_agent = Agent(
    name="network_agent",
    description="网络运维，专注 DNS 与连通性。",
    instruction="仅根据你的知识库回答网络排障问题。不要臆测；必要时建议提交工单。",
    knowledgebase=kb_network,
    model_name=MODEL_NAME,
)

host_agent = Agent(
    name="host_agent",
    description="主机 / 中间件运维。",
    instruction="仅根据你的知识库回答主机/中间件排障问题。必要时建议采集证据或提交工单。",
    knowledgebase=kb_host,
    model_name=MODEL_NAME,
)

app_agent = Agent(
    name="app_agent",
    description="业务应用运维。",
    instruction="仅根据你的知识库回答业务应用排障问题。必要时建议回滚或触发标准化作业。",
    knowledgebase=kb_app,
    model_name=MODEL_NAME,
)

sec_agent = Agent(
    name="sec_agent",
    description="安全运维。",
    instruction="仅根据你的知识库回答安全问题。对可能的失陷给出初步处置建议。",
    knowledgebase=kb_sec,
    model_name=MODEL_NAME,
)

root_agent = Agent(
    name="itops_root",
    description="总控台，把用户意图路由到对应部门子 Agent。",
    instruction=(
        "你是 IT 运维总控台。先判断问题属于 网络/主机/应用/安全 哪一类，"
        "再调用相应子 Agent 获取答案并整合。跨域问题可分别调用多个子 Agent。"
        "涉及动作（如创建工单）时，先说明并征得用户确认，再调用相关工具。"
    ),
    model_name=MODEL_NAME,
    tools=[query_cmdb, create_ticket],
    sub_agents=[network_agent, host_agent, app_agent, sec_agent],
)

runner = Runner(agent=root_agent, app_name="itops_demo", user_id="demo_user")
```

## 第 4 步：运行

```python
import asyncio

if __name__ == "__main__":
    print(
        asyncio.run(
            runner.run(
                messages="生产环境 DNS 解析异常，怎么排查？",
                session_id="itops-1",
            )
        )
    )
```

## 效果

- **跨部门知识统一访问**：一个入口路由到正确的部门知识库；
- **知识隔离**：各子 Agent 只看到自己部门的 Runbook，减少串扰；
- **动作可控**：创建工单等动作在用户确认后才执行。
