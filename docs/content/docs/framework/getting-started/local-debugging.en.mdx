---
title: "Local debugging"
description: "Use the VeADK CLI to visually debug your agent in the browser."
---

VeADK ships with a built-in visual debugging interface that lets you interact with your agent and inspect its execution in the browser, making local development and troubleshooting easier.

## Project Structure

Before launching the debugging interface, make sure your agent directory follows this structure:

```bash
.
└── my_agent
    ├── __init__.py
    └── agent.py
```

`agent.py` should define a `root_agent` instance.

## Launch the Debugging Interface

From the **parent directory** of `my_agent` (i.e., the directory that contains your agent folder), run:

```bash
veadk web
```

Once started, open [http://localhost:8000](http://localhost:8000) in your browser to access the visual debugging interface.

To bind the service to a different host (for example, to allow access from your local network), use the `--host` flag:

```bash
veadk web --host 0.0.0.0
```

<Callout type="info">
  The default port is `8000`. VeADK automatically discovers available agent directories based on your current working directory.
</Callout>

## Using the interface

Once the interface loads:

- Select the agent directory you want to debug from the dropdown in the top-left corner (e.g., `my_agent`);
- The agent is loaded only when you start the first conversation;
- Switch between multiple agents by choosing a different directory name in the dropdown.
