This page combines [short-term memory](/en/docs/framework/memory/short-term) and [long-term memory](/en/docs/framework/memory/long-term) into a single agent — a support assistant that stays coherent within one conversation and remembers a user's preferences when they come back later.

The two kinds of memory have distinct jobs:

- **Short-term memory** holds the current session's context (the latest question, answers already given, tool-call results), keyed by `session_id`.
- **Long-term memory** persists user information (preferences, past requests) across sessions; the agent retrieves it via the `load_memory` tool.

<Callout type="info">
The `local` long-term backend embeds memories, so install the extensions dependency: `pip install "veadk-python[extensions]"`. For production, switch to `viking` or `mem0`.
</Callout>

## Design approach

1. Use a `sqlite` short-term memory to keep multi-turn coherence within a session (persisted, survives restarts).
2. Use a `local` (or `viking` in production) long-term memory to remember preferences across sessions, with `auto_save_session=True` for automatic archiving.
3. Give the agent a ticket-status tool to show memory and tools working together.

## Implementation

### Prerequisites

```bash
pip install "veadk-python[extensions]"
export MODEL_AGENT_API_KEY="<your Ark API key>"
```

### Define a tool

A simple function tool that looks up ticket status (an in-memory dict stands in for a ticketing system):

```python
TICKETS = {
    "T-1001": "In progress: an engineer is assigned, reply expected within 24 hours.",
    "T-1002": "Resolved: the disk expansion is complete.",
}


def query_ticket(ticket_id: str) -> str:
    """Look up the current status of a ticket.

    Args:
        ticket_id: The ticket number, e.g. "T-1001".

    Returns:
        A description of the ticket's current status.
    """
    return TICKETS.get(ticket_id, f"Ticket {ticket_id} not found")
```

### Assemble the agent

Combine short-term memory, long-term memory, and the tool on a single `Agent`:

```python
from veadk import Agent, Runner
from veadk.memory.long_term_memory import LongTermMemory
from veadk.memory.short_term_memory import ShortTermMemory

APP_NAME = "support_demo"
USER_ID = "user-42"


def build_runner() -> Runner:
    short_term_memory = ShortTermMemory(
        backend="sqlite",
        local_database_path="./support_short_term.db",
    )
    long_term_memory = LongTermMemory(backend="local", app_name=APP_NAME)

    agent = Agent(
        name="support_agent",
        description="Enterprise support assistant for inquiries and ticket tracking.",
        instruction=(
            "You are an enterprise support assistant. Within the same session, "
            "remember what the user already told you and avoid repeated questions. "
            "When the user asks about preferences or requests from previous sessions, "
            "use the `load_memory` tool to recall them. To check ticket status, "
            "call the `query_ticket` tool."
        ),
        short_term_memory=short_term_memory,
        long_term_memory=long_term_memory,
        auto_save_session=True,  # archive to long-term memory after each session
        tools=[query_ticket],
    )
    return Runner(
        agent=agent,
        short_term_memory=short_term_memory,
        app_name=APP_NAME,
        user_id=USER_ID,
    )
```

### Run

In the first conversation the user states a preference and checks a ticket (**short-term memory** keeps the session coherent, the **tool** returns ticket status); the second conversation is brand new, and the agent recalls the preference via **long-term memory**:

```python
import asyncio


async def main() -> None:
    runner = build_runner()

    # Session 1: multi-turn context within one session_id + a tool call
    print(
        await runner.run(
            messages="I'm an enterprise user and prefer to be contacted by email. "
            "What's the status of my ticket T-1001?",
            session_id="chat-1",
        )
    )
    print(
        await runner.run(
            messages="What contact method did I mention earlier?",  # references in-session context
            session_id="chat-1",
        )
    )

    # Session 2: a brand-new session, relying on long-term memory
    print(
        await runner.run(
            messages="I'm back with another question. Do you remember how I want to be contacted?",
            session_id="chat-2",
        )
    )


if __name__ == "__main__":
    asyncio.run(main())
```

## Results

- **Short-term memory**: within `chat-1`, the agent directly references the "prefers email" detail the user just gave, without asking again; `query_ticket` returns the live status of `T-1001`.
- **Long-term memory**: in `chat-2` (a brand-new `session_id`), the agent retrieves the preference archived from the previous session via `load_memory` and answers personally.
- **Combined effect**: "in-session continuity + cross-session knowledge accumulation" improves response accuracy, reduces repetitive work, and supports ongoing workflows such as ticket tracking.
