---
title: "长期记忆"
description: "VeADK 长期记忆跨会话保存重要信息，实现个性化、可持续的智能体交互。"
---

长期记忆（Long-term Memory）跨会话、跨时间保存重要信息——用户偏好、任务历史、知识要点或长期状态。[短期记忆](/cn/docs/framework/memory/short-term)只在单次会话内有效，而长期记忆让智能体在**不同会话**之间也能记住事实。

为什么需要它：

- 支持跨会话的连续对话体验；
- 让智能体在多次交互中保留学习成果和用户特定信息；
- 减少重复询问，提升满意度与效率；
- 支撑长期策略优化，如个性化推荐或任务追踪。

## 支持的后端

调试可以用 `local` 后端，生产更建议使用 `viking` 或 `mem0`。

| 后端 | 说明 |
| :----------- | :--------------------------------------------------------- |
| `local` | 内存跨会话记忆，进程退出即清空（仅用于本地调试）。需要安装 `veadk-python[extensions]`。 |
| `viking` | 使用 VikingDB 记忆库作为存储（生产推荐）。 |
| `mem0` | 使用 Mem0 记忆库作为存储（生产推荐）。 |
| `opensearch` | 使用 OpenSearch 作为存储，可持久化与检索。 |
| `redis` | 使用 Redis 作为存储，需支持 RediSearch 功能。 |
| `viking_mem` | 已废弃，设置后自动转为 `viking`。 |

## 初始化

使用长期记忆前，先实例化 `LongTermMemory` 并指定后端，再把它绑定到 `Agent`。把 `long_term_memory` 传给 `Agent` 后，智能体会自动获得一个 `load_memory` 工具，可在运行时检索过往会话。

```python
from veadk import Agent, Runner
from veadk.memory.long_term_memory import LongTermMemory

APP_NAME = "ltm_demo"

# backend="viking" 使用 VikingDB；app_name 用于数据隔离
long_term_memory = LongTermMemory(backend="viking", app_name=APP_NAME)

root_agent = Agent(
    name="ltm_agent",
    instruction="回答用户问题。如果答案可能在过往对话里，使用 `load_memory` 工具检索。",
    long_term_memory=long_term_memory,
)

runner = Runner(agent=root_agent, app_name=APP_NAME)
```

<Callout type="info">
`local` / `viking` / `mem0` 等后端会对记忆做向量化（embedding），需要安装扩展依赖：`pip install "veadk-python[extensions]"`，并配置 embedding 模型（缺省时复用 `MODEL_AGENT_API_KEY`）。
</Callout>

## 记忆管理

### 添加会话到长期记忆

会话结束或达到某个节点时，显式调用 `add_session_to_memory` 把会话数据持久化。它是异步方法，需要 `await`。对于 Viking 后端，这一步会触发数据的向量化。

```python
# 假设 runner 已经完成了一次对话
completed_session = await runner.session_service.get_session(
    app_name=APP_NAME, user_id=USER_ID, session_id=SESSION_ID
)

# 将完整会话归档到长期记忆
await long_term_memory.add_session_to_memory(completed_session)
```

### 检索长期记忆

除了智能体在运行时通过 `load_memory` 自动检索外，你也可以直接调用 `search_memory` 做语义搜索，用于调试或构建自定义 RAG 应用。该方法是异步的，且参数为关键字参数：

```python
response = await long_term_memory.search_memory(
    app_name=APP_NAME,
    user_id=USER_ID,
    query="favorite project",
)
print(response.memories)
```

## 跨会话示例

下面是一个完整、可运行的端到端流程：在会话 #1 里告诉智能体一个事实，开启 `auto_save_session=True` 让 VeADK 自动归档该会话，然后在**全新的**会话 #2 提问——智能体通过检索长期记忆（而非上下文窗口）回忆起这个事实。

这里用 `local` 后端，需要 `pip install "veadk-python[extensions]"`。

```python
import asyncio

from veadk import Agent, Runner
from veadk.memory.long_term_memory import LongTermMemory

APP_NAME = "ltm_demo"
USER_ID = "user-42"


def build_runner() -> Runner:
    long_term_memory = LongTermMemory(backend="local", app_name=APP_NAME)
    agent = Agent(
        name="ltm_agent",
        instruction=(
            "你是个人助理。当用户问起之前告诉过你的事情时，"
            "使用 `load_memory` 工具去回忆。"
        ),
        long_term_memory=long_term_memory,
        auto_save_session=True,  # 每个会话结束后自动写入长期记忆
    )
    return Runner(agent=agent, app_name=APP_NAME, user_id=USER_ID)


async def main() -> None:
    runner = build_runner()

    # 会话 #1：告诉它一个事实。auto_save_session 随后会把它存进长期记忆。
    print(
        "Session 1 ->",
        await runner.run(
            messages="记一下：我对花生过敏，而且我是素食者。",
            session_id="session-1",
        ),
    )

    # 会话 #2：一个全新的会话。智能体必须通过记忆检索来回忆。
    print(
        "Session 2 ->",
        await runner.run(
            messages="帮我推荐一道适合我的菜，要考虑我的饮食限制。",
            session_id="session-2",
        ),
    )


if __name__ == "__main__":
    asyncio.run(main())
```

智能体能在会话 #2 中识别同一用户在会话 #1 留下的偏好，给出连贯、个性化的回答（如推荐一道无花生的素食菜品）。

## 自动保存会话

如上例所示，在初始化 `Agent` 时开启 `auto_save_session=True`，并完成长期记忆配置，VeADK 就会自动把会话保存到长期记忆，无需手动调用 `add_session_to_memory`。

```python
from veadk import Agent
from veadk.memory.long_term_memory import LongTermMemory

long_term_memory = LongTermMemory(backend="viking", app_name="ltm_demo")

agent = Agent(
    name="ltm_agent",
    auto_save_session=True,
    long_term_memory=long_term_memory,
)
```

为避免索引被频繁初始化，VeADK 提供 `MIN_MESSAGES_THRESHOLD` 和 `MIN_TIME_THRESHOLD` 两个环境变量来自定义保存周期。默认在累计 10 条 event 或间隔 60 秒时触发保存；此外，当切换 `session_id` 并发起新问答时，VeADK 会自动把上一个会话的内容保存到长期记忆。
