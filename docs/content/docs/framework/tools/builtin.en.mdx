This document explains how to effectively use VeADK built-in tools (BuiltinTools). These tools provide ready-to-use capabilities (such as web search or a code executor) that give your Agent general-purpose abilities. For example, an Agent that needs to retrieve information from the web can directly use the **web_search** tool with no extra configuration.

## How to use

1. **Import**: import the tool you need from the `veadk.tools.builtin_tools` module.
2. **Configure**: initialize the tool and provide parameters as needed.
3. **Register**: add the tool instance to the Agent's `tools` list.

<Tabs items={['Python', 'Golang']}>
  <Tab value="Python">

```python
from veadk import Agent
from veadk.tools.builtin_tools.web_search import web_search

# Register the tool when initializing the Agent
agent = Agent(tools=[web_search])
```

  </Tab>
  <Tab value="Golang">

```go
import (
    "log"

    veagent "github.com/volcengine/veadk-go/agent/llmagent"
    "github.com/volcengine/veadk-go/tool/builtin_tools/web_search"
    "google.golang.org/adk/agent/llmagent"
    "google.golang.org/adk/tool"
)

func main() {
    webSearch, err := web_search.NewWebSearchTool(&web_search.Config{})
    if err != nil {
        log.Fatalf("NewWebSearchTool failed: %v", err)
        return
    }
    cfg := veagent.Config{
        Config: llmagent.Config{
            Tools: []tool.Tool{webSearch},
        },
    }
    veAgent, err := veagent.New(&cfg)
    if err != nil {
        log.Fatalf("NewLLMAgent failed: %v", err)
        return
    }
}
```

  </Tab>
</Tabs>

Once a tool is registered, the Agent autonomously decides whether to call it based on the **user prompt** and **instruction**. The framework automatically executes the tool when it is called.

<Callout type="warn">
Be sure to read the "Limitations" section at the end of this document.
</Callout>

### Web Search

The `web_search` tool lets the Agent search through the Fusion Information Search API. For details, see the [Fusion Information Search API docs](https://www.volcengine.com/docs/85508/1650263).

<Callout type="warn" title="Additional requirements for the web_search tool">
1. You need to configure a Volcengine AK/SK, or use a temporary StsToken authorized via Volcengine IAM.
2. You need to configure the API Key used for the Agent's reasoning model.
</Callout>

<Tabs items={['Python', 'Golang', 'Environment variables']}>
  <Tab value="Python">

```python title="examples/tools/web_search/agent.py"
from veadk import Agent, Runner
from veadk.tools.builtin_tools.web_search import web_search
from veadk.memory.short_term_memory import ShortTermMemory


app_name = "veadk_app"
user_id = "veadk_user"
session_id = "veadk_session"

agent = Agent(
    name="WebSearchAgent",
    model_name="doubao-seed-1-6-250615",
    description="An agent that can get result from Web Search",
    instruction="You are a helpful assistant that can provide information use web search tool.",
    tools=[web_search],
)
short_term_memory = ShortTermMemory()

runner = Runner(
    agent=agent, short_term_memory=short_term_memory, app_name=app_name, user_id=user_id
)


async def main():
    response = await runner.run(
        messages="What is the weather in Hangzhou today?", session_id=session_id
    )
    print(response)


if __name__ == "__main__":
    import asyncio

    asyncio.run(main())
```

```go title="examples/tools/web_search/agent.go"
package main

import (
	"context"
	"log"
	"os"

	veagent "github.com/volcengine/veadk-go/agent/llmagent"
	"github.com/volcengine/veadk-go/common"
	"github.com/volcengine/veadk-go/tool/builtin_tools/web_search"
	"github.com/volcengine/veadk-go/utils"
	"google.golang.org/adk/agent"
	"google.golang.org/adk/cmd/launcher"
	"google.golang.org/adk/cmd/launcher/full"
	"google.golang.org/adk/session"
	"google.golang.org/adk/tool"
)

func main() {
	ctx := context.Background()
	sessionService := session.InMemoryService()

	cfg := &veagent.Config{
		ModelName:    common.DEFAULT_MODEL_AGENT_NAME,
		ModelAPIBase: common.DEFAULT_MODEL_AGENT_API_BASE,
		ModelAPIKey:  utils.GetEnvWithDefault(common.MODEL_AGENT_API_KEY),
	}
	cfg.Name = "WebSearchAgent"
	cfg.Description = "An agent that can get result from Web Search"
	cfg.Instruction = "You are a helpful assistant that can provide information use web search tool."

	webSearch, err := web_search.NewWebSearchTool(&web_search.Config{})
	if err != nil {
		log.Fatalf("NewWebSearchTool failed: %v", err)
		return
	}

	cfg.Tools = []tool.Tool{webSearch}

	rootAgent, err := veagent.New(cfg)
	if err != nil {
		log.Fatalf("Failed to create agent: %v", err)
	}

	config := &launcher.Config{
		AgentLoader:    agent.NewSingleLoader(rootAgent),
		SessionService: sessionService,
	}

	l := full.NewLauncher()
	if err = l.Execute(ctx, config, os.Args[1:]); err != nil {
		log.Fatalf("Run failed: %v\n\n%s", err, l.CommandLineSyntax())
	}
}
```

### VeSearch

The `vesearch` tool lets the Agent search by calling Volcengine's Online Q&A Agent. For details, see the [Online Q&A Agent](https://www.volcengine.com/docs/85508/1512748).

<Callout type="warn" title="Additional requirements for the vesearch tool">
1. You need to configure a Volcengine AK/SK, or use a temporary StsToken authorized via Volcengine IAM.
2. You need to configure the API Key used for the Agent's reasoning model.
3. You need to configure the Online Q&A Agent's agent ID, obtained after creating an agent in the console ([console address](https://console.volcengine.com/ask-echo/my-agent)). The config item name is `TOOL_VESEARCH_ENDPOINT`.
</Callout>

### Run Code

The `run_code` tool enables the Agent to execute code, allowing the model to perform computation, data processing, or run small programs.

<Callout type="warn" title="Additional requirements for the run_code tool">
1. You need to configure a Volcengine AK/SK, or use a temporary StsToken authorized via Volcengine IAM.
2. You need to configure the API Key used for the Agent's reasoning model.
3. You need to configure an AgentKit Tools Id, see the **AgentKit sandbox tools (Tools)** section.
</Callout>

<Tabs items={['Code', 'Environment variables']}>
  <Tab value="Code">

```python title="examples/tools/run_code/agent.py"
from veadk import Agent, Runner
from veadk.tools.builtin_tools.web_search import web_search

user_id = "veadk_user"
session_id = "veadk_session"

    name="data_analysis_agent",
    description="A data analysis for stock marketing",
    instruction="""
        你是一个资深软件工程师，在沙箱里执行生产的代码， 避免每次安装检查, 
        可以使用python lib akshare 下载相关的股票数据。可以通过web_search工具搜索相关公司的经营数据。如果缺失了依赖库, 
        通过python代码为沙箱安装缺失的依赖库。""",
    tools=[run_code, web_search],
)


short_term_memory = ShortTermMemory()

runner = Runner(
    agent=agent, short_term_memory=short_term_memory, app_name=app_name, user_id=user_id
)


async def main():
    response = await runner.run(messages="阳光电源？", session_id=session_id)
    print(response)


if __name__ == "__main__":
    import asyncio

    asyncio.run(main())
```

  </Tab>
  <Tab value="Environment variables">

The following items must be configured as environment variables:

- `AGENTKIT_TOOL_ID`: the default AgentKit sandbox environment Id, used as the fallback for all sandbox tools
- `AGENTKIT_TOOL_ID_SCRIPT`: the dedicated sandbox environment Id for `run_code`; falls back to `AGENTKIT_TOOL_ID` if unset
- `AGENTKIT_TOOL_ID_SKILLS`: the dedicated sandbox environment Id for `execute_skills`; falls back to `AGENTKIT_TOOL_ID` if unset
- `AGENTKIT_TOOL_ID_OPENCODE`: the dedicated sandbox environment Id for `coding`; falls back to `AGENTKIT_TOOL_ID` if unset
- `AGENTKIT_TOOL_HOST`: the EndPoint for calling Volcengine AgentKit Tools
- `AGENTKIT_TOOL_SERVICE_CODE`: the ServiceCode for calling AgentKit Tools
- `AGENTKIT_TOOL_SCHEME`: switches the protocol for calling AgentKit Tools; allows `http`/`https`, defaults to `https`
- `AGENTKIT_TOP_SCHEME`: switches the protocol for calling AgentKit TOP; allows `http`/`https`, defaults to `https`

Environment variables:

- `MODEL_AGENT_API_KEY`: API Key for the Agent's reasoning model
- `VOLCENGINE_ACCESS_KEY`: Volcengine AccessKey used to call WebSearch

<Callout type="warn" title="Additional requirements for the video_generate tool">

### Lark MCP Tools

The `lark` MCP tool set helps you quickly integrate an AI Agent with Feishu (Lark) open capabilities, enabling Agent-based automation such as Feishu cloud document processing, conversation management, and calendar scheduling.

<Callout type="warn" title="Additional requirements for the lark tool">

### Vod MCP Tool

`vod_tools` lets the Agent perform video editing by calling Volcengine's video cloud MCP. For details, see the [VOD MCP Server](https://github.com/volcengine/mcp-server/blob/main/server/mcp_server_vod/README_zh.md).

<Callout type="warn" title="Additional requirements for vod_tools">
1. You need to configure a Volcengine AK/SK.
2. (Optional) You can configure `TOOL_VOD_GROUPS` to select diverse video editing capabilities. Available values:
    - `edit`: video editing tools
    - `intelligent_slicing`: intelligent slicing tools
    - `intelligent_matting`: intelligent matting tools
    - `subtitle_processing`: subtitle processing tools
    - `audio_processing`: audio processing tools
    - `video_enhancement`: video enhancement tools
    - `upload`: upload-related tools
    - `video_play`: video playback tools
3. (Optional) The tool connection timeout `TOOL_VOD_TIMEOUT`, default 10 seconds.
4. Note: if you need multiple tool groups, join them with commas; if not configured, the default is `edit,video_play`.
5. Note: the video cloud tools do not support Space creation; please create a [video cloud space](https://console.volcengine.com/vod/region:vod+cn-north-1/overview/) in advance in the Volcengine video cloud console.
</Callout>

<Tabs items={['Python', 'Environment variables']}>

```python
import asyncio
from veadk import Agent, Runner
```

## System tools

- `load_knowledgebase`: a knowledge base retrieval tool. When you pass the `knowledgebase` parameter to the Agent, this tool is automatically mounted, and the Agent autonomously decides when to query the knowledge base at runtime.
- `load_memory`: a long-term memory retrieval tool. When you pass the `long_term_memory` parameter to the Agent, this tool is automatically mounted, and the Agent autonomously decides when to query long-term memory at runtime.

See [System Tools](/en/docs/framework/tools/system-tools) for details.

## Others

### AgentKit sandbox tools (Tools)

AgentKit sandbox tools provide a variety of runtime-environment tools that agents need to perform tasks, supporting fast integration and convenient invocation; VeADK also provides more built-in tools. The all-in-one tool set includes Browser, Terminal, and Code runtime environments, and switches automatically based on the task.

**Create a sandbox** following these steps:

1. **Create a sandbox tool**: set a custom sandbox name such as `AIO_Sandbox_xxxx`, choose "all-in-one tool set" as the tool set type, and describe it as "Python code sandbox".

![Create sandbox](/assets/images/tools/agentkittools1.png)
![Create sandbox](/assets/images/tools/agentkittools2.png)

2. **Get the result**: after creation, obtain the sandbox ID for `AIO_Sandbox_xxxx` in the console, e.g. `t-ye8dj82xxxxx`.

![Get sandbox ID](/assets/images/tools/agentkittools3.png)

## Limitations

- Built-in tools rely on related Volcengine services. Before use, make sure the corresponding services are enabled and that AK/SK, API Key, and other authentication is configured.
- Some tools (such as `web_scraper`) are in an invitation-only beta; availability is governed by the console.
- Sandbox tools (`run_code`, `execute_skills`, `coding`, `run_sandbox_agent`) require a correctly configured AgentKit Tool Id, otherwise they cannot be invoked.
