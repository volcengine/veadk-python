---
title: "A2UI (Agent-Driven UI)"
description: "Let an agent reply with declarative UI that a client renders as native components, instead of plain text."
---

VeADK integrates Google's [A2UI](https://a2ui.org), which lets an agent reply with **declarative UI** (cards, rows, forms) rendered by a client as native components, instead of plain text.

The set of components an agent may use is described by a **catalog** — a JSON Schema document that plays two roles at once: the instruction the model reads, and the validator that rejects anything off-catalog.

<Callout type="info">
  A2UI requires the optional `a2ui-agent-sdk` dependency. Install it first: `pip install veadk-python[a2ui]`.
</Callout>

## Enable A2UI

Set `enable_a2ui=True` when you create the agent. This uses the bundled "basic" component catalog:

```python
from veadk import Agent

agent = Agent(enable_a2ui=True)  # uses the bundled "basic" component catalog
```

Once enabled, VeADK injects a `send_a2ui_json_to_client` tool into the agent. The agent returns A2UI JSON by calling that tool, and the client renders the UI from it. The example instruction below tells the model to use A2UI when the answer is naturally visual or structured:

```python title="examples/a2ui_agent/agent.py"
from veadk import Agent

INSTRUCTION = """You are a helpful assistant that can render rich UI.

When the answer is naturally visual or structured (a status card, a summary, a
list of options, a small form), reply by calling the `send_a2ui_json_to_client`
tool with A2UI JSON built ONLY from the components in the provided catalog
(Card, Column, Row, Text, Icon, Button, Divider, ...). Put a `Card` at the root,
lay content out with `Column`/`Row`, and use `Text` with a `variant` for
headings. For everything else, just answer in plain text.
"""

agent = Agent(
    name="a2ui_agent",
    description="Demo agent that replies with A2UI rich UI.",
    instruction=INSTRUCTION,
    enable_a2ui=True,
)

# The Google ADK agent loader requires root_agent.
root_agent = agent
```

To see the result in a browser, use the bundled React frontend — see [VeADK Frontend](/en/docs/framework/frontend/web-ui).

## The bundled basic catalog

`get_basic_catalog()` returns Google's "basic" catalog, ready to use. It provides primitives such as `Card`, `Column`, `Row`, `Text`, `Icon`, `Button`, and `Divider`.

```python
from veadk.a2ui import get_basic_catalog

catalog, examples = get_basic_catalog(version="0.9")
```

| Argument | Description |
| :-- | :-- |
| `version` | A2UI catalog version, `"0.8"` or `"0.9"` (default `"0.9"`, the first to expose `createSurface` / `updateComponents` cleanly). |
| `examples_path` | Optional. Path or glob to few-shot example JSON files, injected into the system prompt alongside the schema. |

## Register custom enterprise components

You can extend the component set. A custom component has two halves that share one **catalog id**:

<Steps>

<Step>

### Backend: declare the catalog

Subclass `BaseA2UICatalog` and point `catalog_path` (and optionally `examples_path`) at your own catalog JSON:

```python
from veadk import Agent
from veadk.a2ui import BaseA2UICatalog

class FinanceCatalog(BaseA2UICatalog):
    version = "0.9"
    catalog_path = "/opt/corp/a2ui/finance_catalog.json"   # defines RevenueChart, etc.
    examples_path = "/opt/corp/a2ui/finance_examples"

agent = Agent(enable_a2ui=True, a2ui_catalog=FinanceCatalog())
```

For full control, override `build()` and return a ready `(A2uiCatalog, examples_str)` tuple yourself.

</Step>

<Step>

### Frontend: provide the renderer

Add a directory under `frontend/src/a2ui/components/<Name>/`; it auto-registers. See [VeADK Frontend](/en/docs/framework/frontend/web-ui#adding-a-custom-enterprise-component).

</Step>

</Steps>

<Callout type="info">
  Every component declared in the catalog must have a matching frontend renderer. Components with no registered renderer fall back to a collapsible JSON view, so a catalog/renderer mismatch never crashes the UI.
</Callout>
