---
title: "A2UI（智能体驱动 UI）"
description: "让智能体返回声明式 UI，由客户端以原生组件渲染，而不仅仅是纯文本。"
---

VeADK 集成了 Google 的 [A2UI](https://a2ui.org)，让智能体可以返回**声明式 UI**（卡片、行列布局、表单等），由客户端以原生组件渲染，而不仅仅是纯文本。

智能体可使用的组件集合由一份 **catalog（目录）** 描述。catalog 是一份 JSON Schema 文档，它同时承担两个角色：作为模型阅读的指令，以及对模型输出进行校验的校验器——任何不在 catalog 中的组件都会被拒绝。

<Callout type="info">
  A2UI 依赖可选的 `a2ui-agent-sdk`。请先安装：`pip install veadk-python[a2ui]`。
</Callout>

## 启用 A2UI

在创建智能体时设置 `enable_a2ui=True` 即可启用。默认使用内置的 “basic” 组件目录：

```python
from veadk import Agent

agent = Agent(enable_a2ui=True)  # 使用内置的 "basic" 组件目录
```

启用后，VeADK 会向智能体注入一个 `send_a2ui_json_to_client` 工具。智能体通过调用该工具返回 A2UI JSON，客户端据此渲染界面。下面的示例提示词引导模型在“答案天然是可视化或结构化”时使用 A2UI：

```python title="examples/a2ui_agent/agent.py"
from veadk import Agent

INSTRUCTION = """You are a helpful assistant that can render rich UI.

When the answer is naturally visual or structured (a status card, a summary, a
list of options, a small form), reply by calling the `send_a2ui_json_to_client`
tool with A2UI JSON built ONLY from the components in the provided catalog
(Card, Column, Row, Text, Icon, Button, Divider, ...). Put a `Card` at the root,
lay content out with `Column`/`Row`, and use `Text` with a `variant` for
headings. For everything else, just answer in plain text.
"""

agent = Agent(
    name="a2ui_agent",
    description="Demo agent that replies with A2UI rich UI.",
    instruction=INSTRUCTION,
    enable_a2ui=True,
)

# Google ADK 的智能体加载器要求暴露 root_agent。
root_agent = agent
```

要在浏览器中查看渲染效果，请使用内置的 React 前端，参见 [VeADK 前端](/cn/docs/framework/frontend/web-ui)。

## 内置 basic 目录

`get_basic_catalog()` 返回 Google 提供的 “basic” 目录，开箱即用。该目录提供 `Card`、`Column`、`Row`、`Text`、`Icon`、`Button`、`Divider` 等基础组件。

```python
from veadk.a2ui import get_basic_catalog

catalog, examples = get_basic_catalog(version="0.9")
```

| 参数 | 说明 |
| :-- | :-- |
| `version` | A2UI 目录版本，可选 `"0.8"` 或 `"0.9"`（默认 `"0.9"`，最早完整暴露 `createSurface` / `updateComponents` 的版本）。 |
| `examples_path` | 可选。few-shot 示例 JSON 的路径或 glob，会与 schema 一起注入系统提示词。 |

## 注册企业自定义组件

企业可以扩展组件集合。一个自定义组件由两个“对半”组成，它们共享同一个 **catalog id**：

<Steps>

<Step>

### 后端：声明组件目录

继承 `BaseA2UICatalog`，将 `catalog_path`（以及可选的 `examples_path`）指向你自己的目录 JSON：

```python
from veadk import Agent
from veadk.a2ui import BaseA2UICatalog

class FinanceCatalog(BaseA2UICatalog):
    version = "0.9"
    catalog_path = "/opt/corp/a2ui/finance_catalog.json"   # 定义 RevenueChart 等组件
    examples_path = "/opt/corp/a2ui/finance_examples"

agent = Agent(enable_a2ui=True, a2ui_catalog=FinanceCatalog())
```

如需完全控制，可重写 `build()` 方法，自行返回 `(A2uiCatalog, examples_str)` 元组。

</Step>

<Step>

### 前端：提供组件渲染器

在 `frontend/src/a2ui/components/<组件名>/` 下新建一个目录，组件会自动注册。详见 [VeADK 前端](/cn/docs/framework/frontend/web-ui#添加企业自定义组件)。

</Step>

</Steps>

<Callout type="info">
  目录中声明的每个组件都必须有一个同名的前端渲染器。若某个组件没有注册渲染器，前端会回退到可折叠的 JSON 视图，因此目录与渲染器不匹配也不会导致界面崩溃。
</Callout>
