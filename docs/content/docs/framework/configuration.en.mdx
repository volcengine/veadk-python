VeADK reads configuration from environment variables, `.env`, and `config.yaml`, so you keep secrets and connection details out of your code. This page is the complete reference of configuration items.

## Priority

VeADK reads environment variables in the following priority order (decreasing):

1. System environment variables (variables preset at the system level)
2. `.env` file environment variables
3. Configuration values in the `config.yaml` file

## Environment variable reference

### Volcengine ecosystem

Unified prefix: `VOLCENGINE_`

| Variable | Description |
| :- | :- |
| `VOLCENGINE_ACCESS_KEY` | Volcengine Access Key |
| `VOLCENGINE_SECRET_KEY` | Volcengine Secret Key |

Corresponding `yaml` format:

```yaml title="config.yaml"
volcengine:
  access_key:
  secret_key:
```

## Runtime dynamic configuration

Beyond environment variables and constants, VeADK also supports dynamically updating agent configuration at runtime (hot updates), managed uniformly by the `DynamicConfigManager`. The dynamic configuration uses a JSON structure containing an `agent` array, which can override fields for one or more agents.

The configurable fields per agent are as follows:

| Field | Description |
| :- | :- |
| `id` | Must match the target Agent's ID |
| `name` | Name used for display and logging |
| `description` | Description used for display and logging |
| `model_name` | Model name registered in Volcengine |
| `instruction` | Prompt defining the agent's behavior and capabilities |

Example structure:

```json
{
  "agent": [
    {
      "id": "...",
      "name": "...",
      "description": "...",
      "model_name": "...",
      "instruction": "..."
    }
  ]
}
```
